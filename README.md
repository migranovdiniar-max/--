# Техническое задание

Данный документ описывает **техническое задание**. Реализация решений в рамках этого README **не требуется**.

Задание состоит из трёх независимых частей:

1. Проверка email-доменов (Python).
2. Мини-интеграция с Telegram.
3. Архитектурная задача (email-аутрич).

---

## 1. Проверка email-доменов

### Цель

Разработать простой скрипт на Python для базовой проверки email-доменов.

### Требования

Скрипт должен:

* принимать на вход список email-адресов;
* извлекать доменную часть каждого email;
* проверять наличие MX-записей домена;
* выводить статус для каждого email-адреса.

### Возможные статусы

* **«домен валиден»** — домен существует и имеет корректные MX-записи;
* **«домен отсутствует»** — домен не существует;
* **«MX-записи отсутствуют или некорректны»** — домен существует, но не принимает почту.

### Формат результата

* один файл `*.py`;
* консольный вывод;
* минимальные инструкции по запуску.

---

## 2. Мини-интеграция с Telegram

### Цель

Реализовать простую интеграцию для отправки текста в Telegram без UI и сложной логики.

### Требования

Решение должно:

* принимать текст из файла формата `.txt`;
* отправлять содержимое файла в выбранный приватный Telegram-чат;
* использовать Telegram-бота;
* работать без пользовательского интерфейса.

### Допустимые варианты реализации

* Python-скрипт;
* n8n-флоу;
* любой минимальный технический подход, обеспечивающий работоспособность.

### Ограничения

* UI не требуется;
* оформление и стиль кода не оцениваются;
* важна только корректная работа.

---

## 3. Архитектурная задача

### Цель

Предложить архитектуру, которая позволит обслуживать **около 1200 email-адресов** для аутрича в условиях:

* нескольких клиентов;
* нескольких направлений рассылок;
* минимальной стоимости инфраструктуры;
* высокой отказоустойчивости.

### Ожидаемое описание

В архитектурном решении необходимо раскрыть:

* общее описание архитектуры;
* используемые сервисы и технологические подходы;
* механизм ротации почтовых ящиков и мониторинга;
* принципы распределения нагрузки;
* основные риски и способы их минимизации;
* примерную оценку стоимости инфраструктуры.

### Формат

* текстовое описание;
* объём: **0,5–1 страница**;
